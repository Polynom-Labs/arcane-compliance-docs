---
title: "Request a disclosure case"
description: "How auditors request scoped access to private transaction data through a case request."
---

Auditors create disclosure case requests when they need access to private transaction data for an investigation. The request stays pending until an application administrator approves it.

## Prerequisites

- Access to an **Application** workspace
- **Case governance — Create cases** (`cases:create`)
- Optional for withdrawal: **Case governance — Withdraw pending case requests** (`cases:withdraw_pending_request`)

## Workflow

<Steps>
  <Step title="Open Case Review">
    Sign in and select your application workspace. In the sidebar, click **Case Review**.
  </Step>
  <Step title="Create case">
    Click **Create case** to open the case request form.
  </Step>
  <Step title="Fill request summary">
    Enter the investigation basis and reason. Choose the **Scope period** (date range) and **Access window** (access from date and duration in days).
  </Step>
  <Step title="Choose disclosure scope">
    Select the disclosure toggles:
    - **Transaction Identifiers**
    - **Sender Information**
    - **Recipient Information**
  </Step>
  <Step title="Assign auditors">
    In **Assigned auditors**, select team members who should access the case if approved. The session user is auto-selected when they appear in the assignable list.
  </Step>
  <Step title="Submit request">
    Click **Submit request**. The case appears in the pending queue until an administrator approves or rejects it.
  </Step>
</Steps>

## Withdraw a pending request

If you created a pending request and no longer need it:

1. Open **Case Review**
2. Find your pending case row
3. Click **Withdraw**

Withdrawal requires **Case governance — Withdraw pending case requests** (`cases:withdraw_pending_request`).

## What happens next

- The request status is **pending**
- Application administrators with **Case governance — Approve case creation** (`cases:approve_creation`) review it under **Disclosure Requests**
- On approval, the system creates an active case with your requested scope and auditor assignments

See [Approve a disclosure request](/auditor-use-cases/approve-disclosure-request) for the administrator workflow.
