Private transfers rely on cryptographic commitments, nullifiers, and ECDH-based private addresses. This page describes the objects involved and how the client SDK helps you derive shared secrets and encrypt application data.

<Warning>
The Poseidon hash implementation in `soroban-privacy-pools` is a prototype for research and educational purposes. It is not audited and exists to keep Circom circuits and Rust/WASM code consistent.
</Warning>

## Cryptographic model

A private coin is represented by:

| Field | Role |
| --- | --- |
| `value` | Amount in stroops |
| `asset[0]`, `asset[1]` | Stellar asset contract id as two field limbs |
| `nullifier` | Secret spent to prevent double-spending |
| `secret` | Derived from deposit ephemeral scalar |
| `sharedSecret[0]`, `sharedSecret[1]` | ECDH key coordinates binding the coin to a recipient |
| `commitment` | Public pool leaf hash |

## Commitment generation

The `CommitmentHasher` template in `circuits/commitment.circom` defines:

```
precommitment = Poseidon(secret, sharedSecret[0], sharedSecret[1])
balanceCommitment = Poseidon(value, asset[0], asset[1])
commitment = Poseidon(nullifier, balanceCommitment, precommitment)
```

The Rust client SDK in `client-sdk/crate/src/coin.rs` implements the same formulas for off-chain coin generation and Merkle witness construction.

## Nullifiers

- The raw `nullifier` stays private in the zero-knowledge witness.
- `nullifierHash = Poseidon(nullifier)` is public when a coin is spent.
- The Soroban contract stores nullifier hashes and rejects reuse.

Use `PrivacyPoolSDK.calculateNullifierHash(nullifier)` to compute the hash from a coin's decimal nullifier string.

## Deposit-side private address flow

1. The recipient publishes a stealth private address encoded as Bech32 with HRP `stpl1`.
2. The depositor samples an ephemeral scalar (`< 2^253` bits).
3. The depositor derives an ephemeral public key and an ECDH shared secret with the recipient's public key.
4. The deposit circuit sets `secret = Poseidon(ephemeralKeyScalar)` and includes the shared secret in the commitment.
5. The ephemeral public key is stored alongside the commitment on-chain.

## Withdrawal-side recovery

1. The recipient holds their private scalar (derived from a Stellar wallet signature).
2. The recipient reads the depositor's ephemeral public key from the on-chain leaf.
3. ECDH produces the same shared secret as on the deposit path.
4. The withdrawal circuit recomputes the commitment, proves Merkle inclusion, and publishes only the nullifier hash.

## Private address generation

1. Build a sign message with `PrivacyPoolSDK.buildStealthAddressSignMessage(stellarAddress)`.
2. The user signs the message with their Stellar wallet (Ed25519).
3. The SDK hashes the signature with SHA-256 and derives a BabyJubJub scalar.
4. The scalar maps to a public point, encoded as a `stpl1` Bech32 address.

```ts
const message = PrivacyPoolSDK.buildStealthAddressSignMessage(stellarAddress);
const signature = await wallet.signMessage(message);
const sdk = await PrivacyPoolSDK.init();
const stealthAddress = await sdk.generateStealthAddressFromStellarSignature(signature);
```

## Install the SDK

```bash
npm install @auditable/privacy-pool-zk-sdk
```

Requirements:

- Node.js `>=19.0.0`, or
- A browser with Web Crypto (`crypto.subtle`)

In the browser, pass WASM and circuit assets to `PrivacyPoolSDK.init()`:

```ts
const sdk = await PrivacyPoolSDK.init({
  wasmBinary: await fetch("/pkg/client_sdk_wasm_bg.wasm").then((r) => r.arrayBuffer()),
  circuitWasm: await fetch("/assets/main.wasm").then((r) => r.arrayBuffer()),
  zkey: await fetch("/assets/main_final.zkey").then((r) => r.arrayBuffer()),
});
```

## Encrypt data for a private address

The SDK provides ECDH helpers. Symmetric encryption uses the platform Web Crypto API. This example derives a shared secret from a random nonce and the recipient's `stpl1` address, then encrypts a JSON payload with AES-GCM.

```ts
import {
  PrivacyPoolSDK,
  decodeStealthAddress,
  generateRandomScalarHex32,
} from "@auditable/privacy-pool-zk-sdk";

const textEncoder = new TextEncoder();

function hexToBytes(hex: string): Uint8Array {
  const normalized = hex.replace(/^0x/i, "");
  return Uint8Array.from(
    normalized.match(/.{1,2}/g)?.map((byte) => Number.parseInt(byte, 16)) ?? [],
  );
}

async function aesKeyFromSharedSecret(sharedSecret: { x: string; y: string }) {
  const material = new Uint8Array([
    ...hexToBytes(sharedSecret.x),
    ...hexToBytes(sharedSecret.y),
  ]);
  const digest = await crypto.subtle.digest("SHA-256", material);
  return crypto.subtle.importKey("raw", digest, "AES-GCM", false, ["encrypt"]);
}

export async function encryptForPrivateAddress(options: {
  recipientPrivateAddress: string;
  payload: unknown;
}) {
  const sdk = await PrivacyPoolSDK.init();
  const recipient = decodeStealthAddress(options.recipientPrivateAddress);
  const randomNonceScalar = generateRandomScalarHex32();
  const ephemeralKey = sdk.ecdhEphemeralPublicKeyFromScalarHex(randomNonceScalar);
  const sharedSecret = sdk.ecdhSharedKey(randomNonceScalar, recipient.x, recipient.y);
  const key = await aesKeyFromSharedSecret(sharedSecret);
  const iv = crypto.getRandomValues(new Uint8Array(12));
  const plaintext = textEncoder.encode(JSON.stringify(options.payload));
  const ciphertext = await crypto.subtle.encrypt({ name: "AES-GCM", iv }, key, plaintext);

  return {
    iv: Array.from(iv),
    ciphertext: Array.from(new Uint8Array(ciphertext)),
    ephemeralKey: sdk.encodeDecodedEphemeralKey(ephemeralKey),
  };
}
```

## Decrypt data as the recipient

The recipient combines their private scalar with the depositor's encoded ephemeral key to recover the shared secret and decrypt.

```ts
import {
  PrivacyPoolSDK,
  type EncodedEphemeralKey,
} from "@auditable/privacy-pool-zk-sdk";

const textDecoder = new TextDecoder();

function hexToBytes(hex: string): Uint8Array {
  const normalized = hex.replace(/^0x/i, "");
  return Uint8Array.from(
    normalized.match(/.{1,2}/g)?.map((byte) => Number.parseInt(byte, 16)) ?? [],
  );
}

async function aesKeyFromSharedSecret(sharedSecret: { x: string; y: string }) {
  const material = new Uint8Array([
    ...hexToBytes(sharedSecret.x),
    ...hexToBytes(sharedSecret.y),
  ]);
  const digest = await crypto.subtle.digest("SHA-256", material);
  return crypto.subtle.importKey("raw", digest, "AES-GCM", false, ["decrypt"]);
}

export async function decryptFromPrivateAddress(options: {
  recipientScalar: string;
  ephemeralKey: EncodedEphemeralKey;
  iv: number[];
  ciphertext: number[];
}) {
  const sdk = await PrivacyPoolSDK.init();
  const ephemeralKey = sdk.decodeDecodedEphemeralKey(options.ephemeralKey);
  const sharedSecret = sdk.sharedSecretFromRecipientPreimage({
    recipientScalar: options.recipientScalar,
    ephemeralKey,
  });
  const key = await aesKeyFromSharedSecret(sharedSecret);
  const plaintext = await crypto.subtle.decrypt(
    { name: "AES-GCM", iv: new Uint8Array(options.iv) },
    key,
    new Uint8Array(options.ciphertext),
  );

  return JSON.parse(textDecoder.decode(plaintext));
}
```

<Note>
These encryption snippets are implementation examples. They are not on-chain protocol primitives. Production integrations should review key derivation, authenticated encryption, and replay protection for your threat model.
</Note>
